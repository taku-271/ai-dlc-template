# .steering/

プロジェクトの設計判断を永続化するディレクトリです。

`docs/inception/` が「何を作るか」を定義するのに対し、`.steering/` は「なぜそう作ったか」の根拠を記録します。
セッションをまたいでも AI が設計の文脈を参照できるようにするための ADR（Architecture Decision Record）置き場です。

| ファイル | 内容 |
|---|---|
| `architecture.md` | アーキテクチャ全体の判断根拠・技術選定の背景 |
| `decisions.md` | 重要な設計判断の記録（ADR形式） |
| `data-models.md` | データモデルの設計意図・制約の理由 |

## 使い方

- 実装を始める前に `.steering/` を読み、設計の意図を把握する
- 新しい技術選定や設計判断を行ったら `decisions.md` に追記する（自動で書き込まず、承認を得てから追記）
- 「なぜ」が自明でない判断はここに残す
