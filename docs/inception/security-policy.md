# セキュリティ・パフォーマンス規約

## 認証・認可
- 全てのmutation（書き込み操作）は認証チェック必須
- 認可チェックはビジネスロジック層で行う（フロントエンドのみでの制御禁止）
- セッション管理はサーバーサイドで行う

## 入力バリデーション
- ユーザーからの全入力はサーバーサイドでバリデーションする
- バリデーションはAPI境界で行う
- エラーメッセージに内部情報（スタックトレース・SQLなど）を含めない

## データ保護
- パスワードはハッシュ化して保存する（平文禁止）
- 機密情報はログに出力しない
- SQLインジェクション・XSS対策を必ず実施する

## パフォーマンス
- N+1クエリは禁止
- 一覧取得にはページネーションを実装する
